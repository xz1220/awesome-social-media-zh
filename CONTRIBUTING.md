# 贡献指南

感谢补充适合 AI Native 创作者和 AI Agent 用户的内容创作资源。

## 基本原则

- 中文说明优先。
- 优先说明资源能放进哪一步 AI 内容工作流：研究、抓取、整理、生成、审校、发布或复盘。
- 优先收录能成为 agent 工具、数据源、上下文、模板、评估标准或自动化节点的资源。
- 不收录刷量、违规采集、侵犯隐私或规避平台规则的资源。
- 不编造工具能力、价格、指标、平台规则或案例表现。
- 不提交未经授权的私密截图、聊天记录、后台数据或用户信息。
- 推荐资源时说明适用场景和限制，不只贴链接。

## 条目格式

优先使用一行说明：

```markdown
- [资源名](https://example.com) - 解决什么问题，适合谁；主要限制或风险；观察日期：YYYY-MM-DD。
```

如果是案例，可以写成：

```markdown
- [账号或内容名称](https://example.com) - 平台；值得学习的点；不适合照搬的点；观察日期：YYYY-MM-DD。
```

## 提交流程

1. 在 `README.md` 中找到最匹配的小节。
2. 按推荐格式补充条目。
3. 确认链接可访问。
4. 确认没有包含私密信息。
5. 运行 `make validate`。
6. 提交 Pull Request。

## 不确定时

如果不确定该放在哪个小节，优先放到最接近的场景分类里，并在 Pull Request 里说明你的判断依据。
