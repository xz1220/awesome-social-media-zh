# awesome-social-media-zh

面向 AI Native 创作者和 AI Agent 用户的中文内容创作资源清单。

这个项目不是传统自媒体工具导航。它关注的是：会使用 ChatGPT、Claude、Codex、Cursor、Claude Code、LangGraph 等工具的人，如何基于 AI Agent 和可验证资料，持续完成选题研究、素材整理、内容生成、多模态资产制作、发布和复盘。

## 内容范围

- AI Native 优先：优先收录能和 AI Agent、代码仓库、自动化工作流、结构化资料库配合使用的资源。
- 创作链路优先：按研究、提取、生成、审校、发布、复盘组织，而不是按工具名堆砌。
- 中文内容优先：最终服务中文创作者、中文平台和中文受众，但可以收录英文工具与文档。
- 可验证优先：工具能力、平台规则、价格和案例观察尽量标注来源与观察日期。
- 安全优先：不收录刷量、违规采集、账号绕控、隐私抓取或平台规避资源。

## 资源区

资源会在后续收集、筛选和确认后加入。现阶段先保留项目定位、收录标准和贡献规则，不预先填充未经讨论的资源或分类。

## 收录标准

一个资源更适合被收录，如果它满足以下条件之一：

- 能帮助 AI Agent 用户完成内容创作链路中的明确步骤，例如检索、抓取、整理、生成、审校、发布或复盘。
- 能作为 agent 的工具、数据源、上下文、模板、评估标准或自动化节点。
- 对中文平台、中文创作者或中文内容出海有明显适配价值。
- 有清晰文档、公开主页、仓库、价格页或可验证来源。
- 已被真实使用过，并有明确的优点、限制和风险说明。

暂不优先收录：

- 传统泛自媒体工具导航，除非它能明确接入 AI Native 工作流。
- 只有营销话术、没有可验证功能说明的工具。
- 需要高风险账号操作、刷量、采集隐私数据或规避平台规则的资源。
- 没有明确用途的泛泛资料。
- 只适合单个私有项目、无法泛化给其他创作者的内容。

## 条目格式

推荐使用一行说明，保持清单可读：

```markdown
- [资源名](https://example.com) - 解决什么问题，适合谁；主要限制或风险；观察日期：YYYY-MM-DD。
```

如果一个资源需要更长说明，可以在条目后补充 1 到 2 行，但不要把 README 变成评测文章。

## 当前阶段

项目处于早期定义阶段，先明确面向 AI Native 创作者和 AI Agent 用户的内容创作资源边界。资源、分类和模板会在后续讨论中逐步加入。

## 本地维护

提交前可以运行轻量校验：

```bash
make validate
```

校验会检查 README 的基础结构、链接格式、重复 URL、常见拼写错误，以及资源条目的观察日期格式。

## 贡献

欢迎补充适合 AI Agent 用户的内容创作资源。提交前请优先说明这个资源能放进哪一步工作流、适合谁、有什么限制。详见 [贡献指南](CONTRIBUTING.md)。
